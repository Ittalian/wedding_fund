# 予測開始日の追加と資金シミュレーションの高度化

ご要望いただいた機能の実装が完了しました。主な変更点は以下の通りです。

## 実装内容

### 1. 「プロポーズ予定日」から「予測開始日」への変更
- [BasicInfoData](file:///Users/itta/dev/dart/wedding_fund/lib/models/basic_info_data.dart) の `proposeDate` フィールドを `forecastStartDate` に変更しました。
- UIのラベルを「プロポーズ予定年月」から「予測開始日」に変更し、入力形式を `yyyy/mm/dd` に変更しました。
- 実在する日付かどうかのバリデーション（例: 2月30日のような入力を防ぐ）を追加しました。

### 2. 収入日・ボーナス日の追加
- [AssetsData](file:///Users/itta/dev/dart/wedding_fund/lib/models/assets_data.dart) に `incomeDate` (1〜31) および `bonusDate` (1〜31) を追加しました。
- UIに入力フィールドを追加し、ボーナス月における該当日が存在するかの厳密なチェック（閏年等も考慮）を実装しました。

### 3. 日付ベースの正確な資金シミュレーション
- 以前は「残り月数 × 月の収入」という概算での計算でしたが、新しい実装では **日付ベースでの発生回数** を正確にカウントするロジックに変更しました。
- **「今日の日付」による判定**: 計算期間に「現在」が含まれる場合、実際の今日の日付(`DateTime.now().day`)が「収入日」より前であればこれからの入金として計算に含め、既に来ていれば現在の貯金に含まれているとみなして計算から除外するロジックを実装しました。
- 資金シミュレーションの起点（初期資金に給与が入り始めるタイミングなど）を、入力された「予測開始日」とするように変更しました。

### 4. 時期計算における貯金残高の表示
- 各費用項目を支払える時期に達した時点での、支出後の「貯金残高」を計算し、時期計算モードのリスト右側に表示するようにしました。

## 確認方法
1. アプリを起動し、[基本情報管理画面](file:///Users/itta/dev/dart/wedding_fund/lib/screens/basic_info_screen.dart) で「予測開始日」を `yyyy/mm/dd` 形式で入力・保存してください。
2. [財産管理画面](file:///Users/itta/dev/dart/wedding_fund/lib/screens/assets_screen.dart) で「平均収入が入る日」と「ボーナスが入る日」を入力・保存してください。
3. ホーム画面の「出費額計算」タブおよび「時期計算」タブにて、収入日をまたいだタイミングで許容出費や達成時期が正確に変動すること、また時期計算の右側に「残高: ¥◯◯」が表示されることをご確認ください。

> 非常に正確なキャッシュフローが計算されるようになったため、今後の支出計画の精度が大きく向上しました！
