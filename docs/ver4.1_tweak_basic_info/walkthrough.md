# 基本情報管理 費用項目入力のUI改善 実装完了

「基本情報管理」画面の費用項目入力エリアを、「出費額計算」モードと「時期計算」モードの2つに分割するUI改修が完了しました。

## 実装内容
- **[MODIFY] [basic_info_screen.dart](file:///Users/itta/dev/dart/wedding_fund/lib/screens/basic_info_screen.dart)**
  - 費用項目リストの上部に `SegmentedButton` を追加し、2つのモードを切り替えられるようにしました。
  - **出費額計算モード**:
    - 目標時期（`targetDate`）の入力・編集を可能にしました。
    - ドラッグでの並び替え操作（`ReorderableListView`）を無効化しました。
    - 入力されている「目標時期」が早い順に、リストが自動的にソートされて表示されるようにしました。時期が同じ場合は優先度順になります。
  - **時期計算モード**:
    - 目標時期（`targetDate`）の入力欄を非表示にしました（既存のデータは裏側で保持されます）。
    - ドラッグで優先度を変更できる従来のUIを提供しています。
  - **バグ防止のロジック改修**:
    - リストがソートされて表示順が切り替わっても、編集や削除が間違った項目に適用されないよう、項目の特定を「インデックス」から「固有のID」ベースに変更しました。

## 確認事項
「基本情報管理」画面を開き、以下の動作をご確認ください。
1. 上部に追加されたセグメントボタンで「出費額計算」と「時期計算」を切り替えられること。
2. 出費額計算モードでは項目の追加・編集時に「目標時期」が入力でき、自動で時期の早い順にソートされること。
3. 時期計算モードでは目標時期が入力できず、ドラッグで優先度の並び替えができること。

これで、ユーザーが目的に応じて直感的に費用項目を整理できるようになりました！
