# 基本情報管理の費用項目入力のUI改善

基本情報管理画面の費用項目の入力部分を、「出費額計算」と「時期計算」の2つのモードに分け、それぞれの用途に適したUIを提供します。

## 背景と目的
現在、費用項目は1つのリストで管理されており、ドラッグでの並び替えと目標時期の入力が混在しています。
ユーザーの目的に応じて表示や操作を最適化するため、`HomeScreen` と同様に `SegmentedButton` でモードを切り替えられるようにします。

## ユーザーへの確認事項 (Open Questions)
> [!NOTE]
> 1. 「時期計算」モードで追加・編集を行う際、目標時期の入力欄は完全に非表示にしますが、内部データとしては既存の目標時期が保持されたままになります（消えません）。これで問題ないでしょうか？
> 2. 「出費額計算」モードでの並び替えについて、目標時期が同じ（または未入力）項目同士は、優先度（時期計算での順序）順に並ぶようにしますがよろしいでしょうか？

## 提案される変更箇所

### 1. `lib/screens/basic_info_screen.dart`
**[MODIFY] `lib/screens/basic_info_screen.dart`**
以下の改修を行います。

- **モード切り替えボタンの追加**:
  - `_InfoMode { expense, timing }` (または類似のenum) を定義し、状態として保持します。
  - 費用項目リストの上部に `SegmentedButton` を配置します。
- **出費額計算モードのUI**:
  - `ListView.builder` を使用し、ドラッグ操作用のアイコン（ハンドル）を非表示にします。
  - 内部で `ExpenseItem` のリストを「目標時期の昇順」に自動ソートして表示します（大元のデータの優先度は維持されます）。
  - 追加・編集ダイアログでは「目標時期」の入力欄を表示します。
- **時期計算モードのUI**:
  - `ReorderableListView.builder` を使用し、ドラッグ操作で優先度を変更できるようにします（従来の動作）。
  - 追加・編集ダイアログでは「目標時期」の入力欄を非表示にします。
- **リスト操作ロジックの修正**:
  - 表示順がデータ順と一致しなくなる（出費額計算モード時）ため、編集・削除時の処理を「インデックス」ベースから「ID」ベースに変更し、意図しない項目が編集・削除されるバグを防ぎます。

## 検証計画 (Verification Plan)
1. 「出費額計算」モードで項目を追加し、目標時期が早い順にリストが自動で並び替わることを確認する。
2. 「出費額計算」モードではドラッグ操作ができないことを確認する。
3. 「時期計算」モードに切り替え、ドラッグで順番が入れ替えられること、追加・編集ダイアログに目標時期欄が出ないことを確認する。
4. モードを切り替えても、編集・削除が正しい項目に対して実行されることを確認する。
