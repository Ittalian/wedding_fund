# 概要とゴール

結婚・同棲に向けて必要な資金から、毎月のやりくり（出費可能額）を算出・管理できるFlutterアプリを作成します。
データベースとしてFirebase(Firestore)を活用し、データの永続化を行います。アプリ実装完了後に、FlutterFire CLIを用いたFirebaseのセットアップを含めた環境構築手順書（セットアップ資料）を作成します。

## 実装方針

### フレームワーク / パッケージ
- アプリケーション: **Flutter**
- データベース: **Firebase (Cloud Firestore)**
- 状態管理: **flutter_riverpod** (直感的で現代的な状態管理)
- ルーティング: 標準の Navigator を使用 (画面数が少ないため)
- UIデザイン: モダンでわかりやすいマテリアルデザイン (Material 3) を適用します。ウェディングを想起させる優しい色合いを想定しています。

### データモデル(Firestoreの設計想定)
アプリユーザーごとのデータとして以下の構造で保存します。シンプルに動作させるため、初期実装では匿名認証もしくは単一のドキュメントに保存する形を想定します。

1. **財産管理 (Assets)**
   - `currentSavings`: 現在の貯金額
   - `monthlyIncome`: 月毎の平均収入
   - `bonusAmount`: ボーナスの金額（1回あたり）
   - `bonusMonths`: ボーナスを受け取る月（例: 6月、12月など。リスト形式）

2. **基本情報 (BasicInfo)**
   - `weddingDate`: 結婚（入籍）予定日
   - `engagementRing`: 婚約指輪費用
   - `weddingRing`: 結婚指輪費用
   - `weddingCeremony`: 結婚式費用
   - `honeymoon`: 新居旅行費用
   - `movingCost`: 新居の契約金
   - `furnitures`: 家財道具のリスト（品名と金額のペアの配列）
   - `savingsGoal`: 追加で残しておきたい目標貯金額

### ロジック（計算式）
1. **必要資金の合計** = 婚約指輪 + 結婚指輪 + 結婚式 + 新居旅行 + 新居契約金 + (家財道具の合計) + 追加貯金目標額
2. **目標貯金額** = 必要資金の合計 - 現在の貯金額
3. **残り月数** = 現在の日付から結婚予定日までの月数（現在より未来の場合）
4. **毎月の必要貯金額** = 目標貯金額 / 残り月数
5. **予定されるボーナス総額** = 現在の日付から結婚予定日までの期間に含まれる「ボーナスの月」の回数 × ボーナス金額
6. **月の出費可能額（やりくり額）** = ((月毎の平均収入 × 残り月数) + 予定されるボーナス総額 - 目標貯金額) / 残り月数
7. **全体の出費可能額** = 月の出費可能額 × 残り月数

### アプリケーションの画面構成
1. **ホーム画面**
   - 算出した「月毎の出費可能額」と「全体の出費額」を大きく見やすく表示
   - 両方の情報（基本情報・財産管理）が未入力の場合や、必須項目がない場合は「情報を入力してください」という案内を表示
   - 財産管理画面・基本情報管理画面それぞれへの遷移ボタン
2. **財産管理画面**
   - 各種収入・貯金額の入力フォームと保存ボタン
   - ボーナスの回数はセレクトボックス(Dropdown)で選択可能に
3. **基本情報管理画面**
   - 各種必要費用の入力フォームと保存ボタン
   - 「家財道具」の動的リスト追加UI（項目追加・削除が可能）
