# ウェディング資金管理アプリ 実装の振り返り

要件に基づき、FlutterとFirebase(Firestore)を用いたウェディング資金管理アプリケーションのソースコードを作成いたしました。

## 実施した変更

1. **プロジェクトの初期化とパッケージ追加**
   - 新規Flutterプロジェクト `wedding_fund` を `/Users/itta/dev/dart/wedding_fund` 配下に作成
   - `flutter_riverpod`, `freezed`, `cloud_firestore` などを追加
2. **データモデルの作成**
   - `AssetsData`: 基本的な貯蓄、月ごとの収入、ボーナス額、およびボーナス月を保持
   - `BasicInfoData`: 結婚予定日と各種必要経費、家財道具の配列(`FurnitureItem`)を保持
3. **ビジネスロジック (StateNotifier)**
   - Riverpodによる状態管理を導入（`app_state_provider.dart`）
   - **計算ロジックのポイント**:
     - *目標貯金額*の計算
     - 現在の日付から結婚予定日までの間で、**ユーザが設定した「ボーナス月」と合致する月が何度訪れるかを正確にカウント**し、予定される将来のボーナス総額を算出。過去のものはカウントしません（フィードバック反映部分）。
     - その上で、今後残された月数と算出した総額をもとに「月の出費可能額（やりくり額）」と「全体の出費可能額」を展開。
4. **UI画面の作成**
   - **ホーム画面**: 予測結果をわかりやすいCardで表示。未入力がある時の警告表示・赤字アラートも搭載。
   - **資産管理画面**: ユーザー入力およびボーナス月を複数選べる（1~12月のチップ選択式）。
   - **基本情報管理画面**: イベント日付選択や家財道具をダイアログから動的に追加できる入力ページ。
5. **セットアップ資料の提供**
   - `setup_guide.md` (アーティファクト) にて、Firebaseプロジェクトとの紐付け（`flutterfire configure`の実行等）の手順を記載。

## 未検証事項とテストの状況
- 今回の開発環境ではユーザーが所有するFirebase Projectへの書き込み接続設定を事前に行うことができないため、Firestoreの通信接続自体（`flutterfire configure`を通じた初期化）は未検証です。
- しかし、アプリ自体のビルドテストおよびコードジェネレーション（`build_runner`等を通じたシリアライズクラスの生成）は完了しています。
- アプリの起動時は、接続がない状態でも一旦はモックとしての空データを初期値として立ち上がるようにしており、Firebaseの初期化エラー時はログ出力される形になっています。

## 次のステップ
提供された `setup_guide.md` の手順に沿って、ご自身のFirebaseコンソール画面からプロジェクトを作成し、「flutterfire configure」コマンドを実行してアプリと紐付けてください！
